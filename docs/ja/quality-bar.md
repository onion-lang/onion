## 行5 — 現在の未解決使い勝手バグ（追跡中）

1. **プリミティブ型の拡張メソッド** — `extension Int { def double(): Int = self * 2 }` とした後、
   `(5).double()` が E0005 を発生させる。`extension String` は動作する。根本原因はまだ確認されていない（boxed-FQCN登録仮説は実証的に反証された）。
2. **クラスメソッドから呼ばれるトップレベル関数** — クラスメソッド内で `helper(21)` とすると E0005 が発生する。クリーンな修正は、トップレベルの `val`/`var` と関数をすべて static にすること（static field/method の TypedAST ノードはすでに存在する）。残る障害は、static 関数本体の裸識別子が static field にフォールバックしないこと。

両方が 0 になり、かつ行3/4が満たされたときに基準はクリアされます。
